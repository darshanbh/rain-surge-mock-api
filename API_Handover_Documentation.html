3. API Endpoints Reference

Method Endpoint Description / Usage
GET / Service health check / online verification.
GET /api/zones Returns status, active weather, and agent counts for all zones (Bangalore, Chennai, etc.).
POST /api/quotation Computes quotation and applies rain surge multiplier based on active weather.
GET /api/app-status Returns UI flags to toggle rain warnings on the mobile app or web headers.
POST /api/zone/weather Changes simulated weather state for a specific zone.
POST /api/scenario/reset Resets all zone weather, rules, and cache values back to default.

5. Testing Simulated Failures

Append the fail query parameter to any request URL to test client-side resilience and error screens:

  • ?fail=timeout → Returns 408 Request Timeout
  • ?fail=http500 → Returns 500 Internal Server Error
  • ?fail=http429 → Returns 429 Too Many Requests
